# 🧩 New Idea — Intake Checklist

For each new idea: make a folder `design-refs/<short-idea-name>/`, copy this file
into it as `checklist.md`, fill it out (roughly is fine), and drop images beside it.
Then tell Claude: *"new idea — see design-refs/<short-idea-name>/"*.

```
=== NEW IDEA FOR CLAUDE CODE ===

1. THE IDEA (1–3 sentences, plain words):
   What is it? Who is it for?

2. THE JOB IT DOES:
   What should the user be able to DO with it?
   (e.g. "record a sale", "see today's total")

3. SCREENS / PARTS (just list them):
   -
   -

4. REFERENCE IMAGES:
   In this idea's folder  →  filenames: __________
   (or "none, just text")

5. LOOK & FEEL:
   Any colors, vibe, or app you want it to resemble?

6. MUST-HAVE vs NICE-TO-HAVE:
   Must:
   Later:

7. WHICH APP:
   New app, or part of an existing one? (eShaqo / mister-a / new)

8. ANYTHING I'M UNSURE ABOUT:
   (questions or "you decide")
```

---

## The workflow, in one line
**Vision → Plan → Build → Run & Test → Fix → Commit → Push** — repeated in small loops.

## Tips to pull your weight
- **Start small.** One screen or feature per idea.
- **Words first, polish later.** "You decide the rest" is a valid handoff.
- **You're the tester.** Run it and tell me what feels wrong.
- **Trust Git.** Once something works, it's saved — you can't break everything.
